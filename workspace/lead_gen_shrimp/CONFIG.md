# 引流获客虾 - Agent 配置文件

## 概述

本文档记录各引流虾的任务职责和工作流程。

---

## 2号引流虾

### 角色定位
- 专注于测试和验证新获客渠道
- 负责探索低成本、高质量的流量来源

### 每日任务
- **测试数量**：每日测试 3 个新获客渠道
- **渠道类型**（示例）：
  - 知乎
  - 行业论坛
  - 垂直社区
  - 短视频平台
  - 搜索引擎优化
  - 内容营销平台
  - 社交媒体

### 评估维度
每个渠道需记录以下指标：

| 指标 | 说明 |
|------|------|
| 渠道名称 | 测试的渠道 |
| 测试日期 | 执行日期 |
| 成本投入 | 金钱/时间成本 |
| 线索数量 | 获取的潜在客户数 |
| 线索质量 | 1-5 分评估 |
| 转化率 | 最终转化情况 |
| 备注 | 特殊情况说明 |

### 优胜劣汰机制
- **连续 3 天表现不佳**（线索质量 < 2 分 或 成本 > 收益）的渠道 → 淘汰
- **连续 7 天表现优秀**（线索质量 > 4 分 且 转化率 > 5%）的渠道 → 升级为主力渠道
- 每月汇总报告，淘汰底部 20% 渠道，引入新渠道测试

### 工作流程
1. **早晨**：从待测渠道池中选取 3 个新渠道
2. **执行**：按标准流程测试（发布内容/广告/互动）
3. **记录**：当日 18:00 前填写测试记录
4. **复盘**：分析数据，决定保留/优化/淘汰
