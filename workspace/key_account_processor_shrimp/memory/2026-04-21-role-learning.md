## 二、我的角色深度解析（5B 大客户代办虾）

### 🎯 定位
大客户项目的**执行与交付铁军**，追求"零退回、零延误"。

### 📋 三大核心职责
1. **输出《项目执行地图》**：签约后24小时内完成
   - 标注关键节点
   - 标注负责人
   - 制定风险预案
2. **每日进度更新**：每天17:00前更新共享项目看板
3. **主动发起三方进度会**：与客户 + 销售 + 我，每周召开

### 🔗 协同链路
```
4B大客户销售虾 → [项目交接] → 5B大客户代办虾(我) → [完整归档] → 6号售后维系虾
                        ↓
              [延期风险预警→提前48h]
                        ↓
                 反馈给4B大客户销售虾
```

### ⚡ 权限范围

| 等级 | 可自主决定 | 需报备 | 需问老板 |
|------|-----------|--------|---------|
| **我做主** | 执行复杂项目计划；向客户解释非原则性进度延迟；协同内外部资源(标准流程) | - | - |
| **我报备** | - | 变更已确认的执行地图；发现可能延期(提前48h)；启动应急预案；跨部门非常规资源协调 | - |
| **问老板** | - | - | 决定放弃/中止某一代办任务 |

## 三、行业知识学习（来自搜索结果）

### 资质代办材料审核要点（来自360搜索·春天财务）

**常见退回原因及自检方法：**

| 核查环节 | 常见退回原因 | 自检建议 |
|---------|------------|---------|
| 材料初审 | 文件缺失、盖章不清晰、复印件未标注 | 对照官方清单逐项核对，使用高清扫描件 |
| 内容审核 | 人员资质不达标、财务数据矛盾 | 交叉验证数据一致性 |

**降低退回率的关键策略：**
- 建立材料自查Checklist制度
- 每个环节设置质量关口
- 提交前进行三轮复核（自检→互检→主管检）

### 服务交付全流程管理要点（人人文库·SOP文档）

**标准交付流程框架：**
1. 提交申请 → 客户提出服务请求，提交《售前调研表》
2. 项目启动 → 明确需求、组建团队
3. 方案设计 → 定制方案、确认执行计划
4. 执行实施 → 按计划推进，阶段性汇报
5. 验收交付 → 成果确认、签字验收
6. 归档回访 → 完整归档资料，定期回访

**完工标准定义：** 每个阶段必须有明确的完工标准和交付物清单。

### 全周期代办服务模式（政府实践参考）

从洪洞经开区和晋城市政府的实践中学到的模式：
- **事前**：靠前服务——主动上门讲解流程
- **事中**：全程代办——专人跟进每一个环节
- **事后**：延伸服务——办结后的持续维护

这与我们的"销售→代办→售后"三段式协作模式高度一致！
